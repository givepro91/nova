# Nova v2 — 포지셔닝 / 랜딩 브리프

> 랜딩 페이지를 만드는 에이전트를 위한 **메시징 가이드**. 최종 카피가 아니라, *무엇을·왜·어떻게 말할지*의 기준. 톤은 Nova의 철학(절제·정직)을 그대로 따른다 — 과장 금지.

---

## 0. 한 줄

**Nova는 Claude Code 세션이 매번 리셋되지 않고 *복리(compounding)*로 똑똑해지게 만드는, 단일 경량 플러그인이다.** 정직 · git-native · 저(低)컨텍스트-세금. 한 줄 설치.

---

## 1. ⚠️ 가장 중요한 규칙 — v1과 **별개**다

랜딩은 v2를 **그 자체의 제품**으로 말한다. "v1의 다음 버전/업그레이드"로 프레이밍하지 **않는다**.

- **v1 (`TeamSPWK/nova`)** — 무거운 "AI Agent Ops 프레임워크"(5기둥·품질게이트·NOVA-STATE·세션 규칙 주입·수십 커맨드/에이전트·mcp-server·자기진화). 회사(SpaceWalk) 맥락에서 태어남. 회사용으로 유지될 수도 있고 아닐 수도 있다 — **그건 별개 트랙**.
- **v2 (`givepro91/nova`)** — 개인·공개·**lean**한 단일 플러그인. 다른 레포, 다른 org, 다른 철학, 다른 청중.

같은 이름의 계보일 뿐, **다른 물건**이다. v1의 마케팅 언어(5기둥·"Agent Ops 프레임워크")를 v2에 상속시키지 말 것. v2는 *작고 정직한 도구*로 선다.

---

## 2. 왜 v1을 내려놓았나 (정직하게 — 이게 스토리의 힘이다)

숨길 약점이 아니라, **솔직해서 신뢰를 주는** 부분이다.

- v1은 빠르게 움직이는 플랫폼(Claude Code) 위에 얹은 **무거운 프레임워크**였다. 그 핵심 가치(오케스트레이션 + 품질 게이팅)를 **omc(oh-my-claudecode)와 Claude 본체가 따라잡아 흡수**했다 — 차별점이 증발.
- 동시에 v1 자체 메커니즘이 마찰이 됐다: NOVA-STATE 갱신의 토큰 병목, 다수 상시-on 규칙/플러그인의 **컨텍스트 세금**, 유지보수 부담. 결국 방치됐다.
- **교훈**: 빠른 기반 위에 무거운 레이어를 지으면, 기반이 따라잡는 순간 존재 이유가 사라진다. 이건 실패가 아니라 *플랫폼이 성숙한 자연스러운 결과*다.

→ 메시지: "우리는 도구가 플랫폼에 흡수되는 걸 인정했고, **버릴 건 버리고 살릴 하나만 살려** 다시 지었다."

---

## 3. v2는 무엇이고 왜 만들었나

- 살릴 자산은 **하나**였다: **적대적 분리(Generator–Evaluator)** — v1에서 실전으로 런타임 버그를 잡아낸, 검증된 패턴.
- 시장을 적대적으로 분석해 **흡수 불가능한 niche**를 찾았다: **세션 정직성**(completion theater — 에이전트가 몰래 코너 컷하고 "done"이라 보고하는 것). 자율성(Auto Mode·subagent)이 오를수록 **가치가 커지고**, 플랫폼은 *자기 제품이 거짓말한다는 게이트*를 구조적으로 만들지 않는다.
- **철학 전환**: v1은 *doer*(에이전트를 오케스트레이션 — 플랫폼/omc와 경쟁). v2는 ***keeper***(에이전트 *주위*의 상태·규칙·기록·검증 — 플랫폼이 점유하지 않은 레이어).
- 형태: **단일 `nova` 플러그인**, 세션을 원료로 한 **5개 loop**.
  - **규칙**(`/claude-md`) · **학습**(`/learn`) · **연속성**(`/handoff`) · **검증**(`/gate`) · **기록**(`/document`)

---

## 4. 장점 / 차별점 (랜딩의 "왜 좋은가")

1. **복리(compounding)** — 매 세션이 다음 세션을 개선한다: 실수→규칙(`/learn`), 열린 작업→연속성(`/handoff`), 완료→정직성 감사(`/gate`), 한 일→durable 기록(`/document`). 리셋되지 않는다.
2. **흡수 불가 차별점 = 정직성 게이트** — `/gate`가 **독립 검증자**(별도 컨텍스트)를 띄워 세션의 *주장(claim)*을 *증거(evidence)*에 대조한다: 테스트 진짜 돌렸나? 범위 지켰나? 미검증을 done으로 포장했나? (diff만 보는 코드리뷰가 못 잡는 것.)
3. **Lean — 저컨텍스트-세금** — 단일 플러그인, 훅은 전부 **opt-in**(안 켜면 조용). v1의 상시-on 무거움과 정반대.
4. **정직 · git-native · portable** — 회사 내부 도구에 **0 의존**(완전 self-contained), git이 진실원, 한 줄 설치.
5. **엄격하게 검증돼 만들어짐** — 설계가 codex(일반+적대) 2리뷰 + 공식 docs 검증 + code-review를 거쳤고, **자기 자신의 `/gate`로 dogfood**한다. (도구가 자기 정직성 기준을 스스로 통과.)

---

## 5. 계속 업데이트되나 — 그렇다, 단 lean하게

- **유지·진화된다.** 단, v1식 무거운 자동진화(스캐너·ledger·cron)는 **버렸다.** `/nova-dev`라는 *한 자리(one-sitting) 루프*로, **실제 신호가 있을 때만**(Claude Code가 변하거나 구체적 개선거리가 있을 때) 갱신한다 — 스케줄 강박 없음.
- semver + GitHub Releases. 현재 **v0.2.1**. 개인 프로젝트로 지속.
- 메시지: "작게, 그러나 살아있게 — 플랫폼이 변하면 따라가고, 아니면 가만히 둔다."
